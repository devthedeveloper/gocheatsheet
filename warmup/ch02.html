Survive a Restart

from a slice in memory to a real database in one file — SQLite

In this chapter you'll learn ✍️

  • why an in-memory app forgets everything on restart
  • SQLite: a whole database in a single file, no install
  • the three moves: Exec, QueryRow, and the Query row-loop
  • splitting the project into main.go + store.go

Feel the problem first

Start last chapter's server, post a couple of links, then stop it (Ctrl+C) and start it again. Everything's gone. Your links lived in a Go slice, and a slice lives in memory, and memory evaporates when the process exits. Real apps need to remember. Enter the database.

Why SQLite (and this driver)

Postgres and MySQL are separate servers you install and run. SQLite is just a file — linkboard.db — and the engine runs inside your program. Nothing to install, and it's the same database inside your phone and browser. We use the modernc.org/sqlite driver, which is pure Go, so it builds anywhere with no C compiler. One command adds it:

terminal
$ go get modernc.org/sqlite
$ go mod tidy

A new file: store.go

All the database code goes in its own file, store.go. Keeping data code apart from web code is a habit that pays off forever (the Reddit book leans on it hard). Here's the whole file — notice every function is small:

store.go 
package main

import (
	"database/sql"

	_ "modernc.org/sqlite" // registers the "sqlite" driver with database/sql
)

// Link is one submission on the board. It gained a CreatedAt field now that a
// real database can stamp the time for us.
type Link struct {
	ID        int64  `json:"id"`
	Title     string `json:"title"`
	URL       string `json:"url"`
	Votes     int64  `json:"votes"`
	CreatedAt string `json:"created_at"`
}

// db is the one shared connection pool. A package-level variable is fine for an
// app this small; the Reddit book shows how bigger apps pass it around instead.
var db *sql.DB

// openDB connects to the SQLite file and creates the links table if it's not
// there yet. Call it once, at start-up.
func openDB(path string) error {
	var err error
	db, err = sql.Open("sqlite", path+"?_pragma=busy_timeout(5000)")
	if err != nil {
		return err
	}
	if err := db.Ping(); err != nil {
		return err
	}

	_, err = db.Exec(`
        CREATE TABLE IF NOT EXISTS links (
            id         INTEGER PRIMARY KEY AUTOINCREMENT,
            title      TEXT    NOT NULL,
            url        TEXT    NOT NULL,
            votes      INTEGER NOT NULL DEFAULT 0,
            created_at TEXT    NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ','now'))
        )`)
	return err
}

// insertLink saves a new link and returns it, fully filled in by the database
// (id and created_at come back via RETURNING).
func insertLink(title, url string) (Link, error) {
	var l Link
	err := db.QueryRow(`
        INSERT INTO links (title, url)
        VALUES (?, ?)
        RETURNING id, title, url, votes, created_at`,
		title, url,
	).Scan(&l.ID, &l.Title, &l.URL, &l.Votes, &l.CreatedAt)
	return l, err
}

// listLinks returns every link, newest first.
func listLinks() ([]Link, error) {
	rows, err := db.Query(`
        SELECT id, title, url, votes, created_at
        FROM links
        ORDER BY id DESC`)
	if err != nil {
		return nil, err
	}
	defer rows.Close()

	links := []Link{}
	for rows.Next() {
		var l Link
		if err := rows.Scan(&l.ID, &l.Title, &l.URL, &l.Votes, &l.CreatedAt); err != nil {
			return nil, err
		}
		links = append(links, l)
	}
	return links, rows.Err()
}

Three database moves show up here, and they're all the SQL-plumbing you need for the rest of the book:

MoveWhenSeen in
db.Exec(...)run SQL that returns no rows (like CREATE TABLE)openDB
db.QueryRow(...).Scan(...)get exactly one row backinsertLink
db.Query(...) + loopget many rows backlistLinks
  • The blank import _ "modernc.org/sqlite" — the underscore means “run this package's setup (it registers the "sqlite" driver) but I won't call it directly.” Your code only ever talks to the standard database/sql. (The Reddit book's chapter 5 goes deeper if you're curious.)
  • RETURNING in insertLink hands back the columns the database filled in — the auto id and the created_at timestamp — in the same round trip. No second query.
  • The row-loop in listLinks: defer rows.Close(), scan each row, then check rows.Err() at the end. You'll write this exact shape countless times.
Two row-loop habits worth burning in

defer rows.Close() right after a successful Query — forget it and you leak database connections.
return links, rows.Err() at the end — the loop stopping could mean “no more rows” or “something broke mid-read”; only rows.Err() tells you which.

main.go gets simpler

The handlers now just call the store, and the slice and mutex are gone — the database handles concurrent access for us. Here's the updated main.go:

main.go 
// linkboard, chapter 2: the links now live in a SQLite file, so they survive a
// restart. The in-memory slice and its mutex are gone — the database handles
// concurrent access for us.
package main

import (
	"encoding/json"
	"log"
	"net/http"
)

// listLinksHandler — GET /links
func listLinksHandler(w http.ResponseWriter, r *http.Request) {
	links, err := listLinks()
	if err != nil {
		http.Error(w, "could not load links", http.StatusInternalServerError)
		return
	}
	w.Header().Set("Content-Type", "application/json")
	json.NewEncoder(w).Encode(links)
}

// createLinkHandler — POST /links   Body: {"title": "...", "url": "..."}
func createLinkHandler(w http.ResponseWriter, r *http.Request) {
	var input struct {
		Title string `json:"title"`
		URL   string `json:"url"`
	}
	if err := json.NewDecoder(r.Body).Decode(&input); err != nil {
		http.Error(w, "body must be valid JSON", http.StatusBadRequest)
		return
	}

	link, err := insertLink(input.Title, input.URL)
	if err != nil {
		http.Error(w, "could not save link", http.StatusInternalServerError)
		return
	}
	w.Header().Set("Content-Type", "application/json")
	w.WriteHeader(http.StatusCreated) // 201
	json.NewEncoder(w).Encode(link)
}

func main() {
	if err := openDB("linkboard.db"); err != nil {
		log.Fatal(err)
	}
	log.Println("database ready: linkboard.db")

	mux := http.NewServeMux()
	mux.HandleFunc("GET /links", listLinksHandler)
	mux.HandleFunc("POST /links", createLinkHandler)

	log.Println("linkboard listening on http://localhost:4000")
	log.Fatal(http.ListenAndServe(":4000", mux))
}

Peek inside the file

linkboard.db is a real SQLite database. Open it with any SQLite browser (like DB Browser for SQLite) and watch rows appear as you post. Add *.db to a .gitignore — databases don't belong in git.

Pin to your brain 📌

  • In-memory = forgotten on restart. A database = remembered.
  • SQLite is one file, engine inside your process; the modernc driver needs no C compiler.
  • Three moves: Exec (no rows), QueryRow().Scan() (one row), Query + loop (many).
  • Row loop: defer rows.Close(), scan, then rows.Err().
  • Data code lives in store.go, apart from the web code.
