Build Your First Go API

a gentle, 5-chapter warm-up — you'll build linkboard, a tiny link-sharing board, in an evening. Then the Reddit book will feel easy.

~180 lines of Go  ·  one file grows into four  ·  every chapter runs  ·  no experience needed

Who this is for 🐣

  • you know a little Go (variables, structs, functions, if err != nil)
  • you've never built a web API — or you tried the Reddit book and it felt like a lot
  • you want a win today: a real running server you understand top to bottom

Rusty on the Go itself? Keep the cheatsheet open in the next tab.

What you'll build

linkboard is the simplest social site that still feels real: a single page where anyone can post a link and upvote, with the top links floating up — think a one-room Hacker News. No logins, no comments, no fuss. By the last page it's a real JSON API and a little web page, all served by one Go program you wrote yourself.

The five chapters

chapter 01 One File, Whole API ~60 lines, links kept in memory. You'll have a working POST/GET API before your coffee cools. chapter 02 Survive a Restart Your links vanish when the server stops. Give them a real home: one SQLite file, ~40 changed lines. chapter 03 Votes & Sorting Upvotes with one UPDATE, and ?sort=top vs new — the heart of every link aggregator. chapter 04 Be a Good API + a Web Page Tidy JSON errors, a little validation, and a 30-line frontend served right from Go. The “I built a website” moment. chapter 05 Ship It + the Bridge Build one binary, then a map from every linkboard idea to its grown-up form in the Reddit book. Your on-ramp.
How to read this

Type the code, run every chapter. Each chapter ends with real curl commands and the exact output to expect. Stuck? Every chapter has a complete, compiling copy of the project at that point in code/checkpoints/ — copy the folder and keep going.

no frameworks, no ORM, no magic — just Go's standard library and one tiny SQLite driver ✍️
