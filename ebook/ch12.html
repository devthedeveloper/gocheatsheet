Graceful shutdown — paying chapter 3's IOU

Right now, Ctrl+C (or a redeploy sending SIGTERM) kills the process mid-request — a user's half-written comment just… evaporates. The fix: catch the signal, stop accepting new requests, and give in-flight ones ten seconds to finish. Here's main.go in its final form:

cmd/api/main.go — final 
// gopherit — a Reddit-clone REST API built with Go's standard library.
// This is the companion project for the "Go + REST: Build Your Own Reddit"
// handwritten-notes ebook.
package main

import (
	"context"
	"errors"
	"flag"
	"fmt"
	"log/slog"
	"net/http"
	"os"
	"os/signal"
	"syscall"
	"time"

	"gopherit/internal/store"
)

const version = "1.0.0"

// config holds everything tweakable from the outside world.
type config struct {
	port int
	env  string
	dsn  string
}

// application is the "toolbox" every handler can reach: config, a logger
// and the data layer. Handlers hang off it as methods.
type application struct {
	config config
	logger *slog.Logger
	store  *store.Store
}

func main() {
	var cfg config
	flag.IntVar(&cfg.port, "port", 4000, "API server port")
	flag.StringVar(&cfg.env, "env", "development", "Environment (development|production)")
	flag.StringVar(&cfg.dsn, "dsn", "gopherit.db", "SQLite database file")
	flag.Parse()

	logger := slog.New(slog.NewTextHandler(os.Stdout, nil))

	st, err := store.Open(cfg.dsn)
	if err != nil {
		logger.Error("cannot open database", "error", err)
		os.Exit(1)
	}
	defer st.Close()
	logger.Info("database ready", "dsn", cfg.dsn)

	app := &application{
		config: cfg,
		logger: logger,
		store:  st,
	}

	srv := &http.Server{
		Addr:         fmt.Sprintf(":%d", cfg.port),
		Handler:      app.routes(),
		IdleTimeout:  time.Minute,
		ReadTimeout:  5 * time.Second,
		WriteTimeout: 10 * time.Second,
	}

	// Graceful shutdown: when Ctrl+C (SIGINT) or SIGTERM arrives, stop
	// accepting new requests and give in-flight ones 10s to finish.
	shutdownErr := make(chan error)
	go func() {
		quit := make(chan os.Signal, 1)
		signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
		sig := <-quit

		logger.Info("shutting down", "signal", sig.String())
		ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
		defer cancel()
		shutdownErr <- srv.Shutdown(ctx)
	}()

	logger.Info("starting server", "addr", srv.Addr, "env", cfg.env, "version", version)

	err = srv.ListenAndServe()
	if !errors.Is(err, http.ErrServerClosed) {
		logger.Error("server error", "error", err)
		os.Exit(1)
	}
	if err := <-shutdownErr; err != nil {
		logger.Error("graceful shutdown failed", "error", err)
		os.Exit(1)
	}
	logger.Info("server stopped gracefully")
}

The choreography, step by step:

  1. A background goroutine parks on <-quit, asleep until SIGINT/SIGTERM arrives.
  2. On a signal it calls srv.Shutdown(ctx): the listener closes (no new connections), in-flight requests get up to 10s to complete (that's the context timeout).
  3. Meanwhile ListenAndServe returns http.ErrServerClosed — for once, an error that means everything went right — so we treat it as the happy path and wait on the channel for Shutdown's verdict.

Tests that hit the real thing

Unit tests are nice; for an API, integration tests earn their rent: spin up the entire app — router, middleware, handlers, store, a throw-away SQLite file — and walk through it like a user would. Go's net/http/httptest makes this almost unfairly easy:

cmd/api/api_test.go 
// newTestServer spins up the whole app against a throw-away database.
// t.TempDir() is deleted automatically when the test finishes.
func newTestServer(t *testing.T) *httptest.Server {
	t.Helper()

	st, err := store.Open(filepath.Join(t.TempDir(), "test.db"))
	if err != nil {
		t.Fatal(err)
	}
	t.Cleanup(func() { st.Close() })

	app := &application{
		config: config{env: "test"},
		logger: slog.New(slog.NewTextHandler(io.Discard, nil)), // silence logs
		store:  st,
	}

	ts := httptest.NewServer(app.routes())
	t.Cleanup(ts.Close)
	return ts
}
cmd/api/api_test.go (continued) 
// do sends a JSON request and decodes the JSON response into a map.
func do(t *testing.T, ts *httptest.Server, method, path, token string, body any) (int, map[string]any) {
	t.Helper()

	var buf bytes.Buffer
	if body != nil {
		if err := json.NewEncoder(&buf).Encode(body); err != nil {
			t.Fatal(err)
		}
	}

	req, err := http.NewRequest(method, ts.URL+path, &buf)
	if err != nil {
		t.Fatal(err)
	}
	req.Header.Set("Content-Type", "application/json")
	if token != "" {
		req.Header.Set("Authorization", "Bearer "+token)
	}

	res, err := ts.Client().Do(req)
	if err != nil {
		t.Fatal(err)
	}
	defer res.Body.Close()

	var got map[string]any
	if err := json.NewDecoder(res.Body).Decode(&got); err != nil {
		t.Fatalf("decoding response from %s %s: %v", method, path, err)
	}
	return res.StatusCode, got
}

With those two helpers, the test reads like a story — register, get rejected as a duplicate, log in, hit a locked door without a token, then build a community, post, comment, reply and vote, checking status codes and payloads at every step. Here's the heart of it (the full file is in the repo):

cmd/api/api_test.go — the story so far 
// TestHappyPath walks the whole gopherit story: register, log in, create
// a community, post, comment, reply and vote.
func TestHappyPath(t *testing.T) {
	ts := newTestServer(t)

	// Register.
	status, got := do(t, ts, "POST", "/api/v1/users", "", map[string]any{
		"username": "gopher",
		"email":    "gopher@example.com",
		"password": "secret-password",
	})
	if status != http.StatusCreated {
		t.Fatalf("register: want 201, got %d (%v)", status, got)
	}

	// Duplicate registration must be rejected.
	status, _ = do(t, ts, "POST", "/api/v1/users", "", map[string]any{
		"username": "gopher",
		"email":    "gopher@example.com",
		"password": "secret-password",
	})
	if status != http.StatusConflict {
		t.Fatalf("duplicate register: want 409, got %d", status)
	}

	// Log in.
	status, got = do(t, ts, "POST", "/api/v1/tokens", "", map[string]any{
		"email":    "gopher@example.com",
		"password": "secret-password",
	})
	if status != http.StatusCreated {
		t.Fatalf("login: want 201, got %d (%v)", status, got)
	}
	token, _ := got["token"].(string)
	if token == "" {
		t.Fatal("login response did not include a token")
	}

	// Creating a subreddit without a token must fail.
	status, _ = do(t, ts, "POST", "/api/v1/subreddits", "", map[string]any{
		"name": "golang", "title": "The Go community",
	})
	if status != http.StatusUnauthorized {
		t.Fatalf("anonymous create subreddit: want 401, got %d", status)
	}

	// With the token it succeeds.
	status, got = do(t, ts, "POST", "/api/v1/subreddits", token, map[string]any{
		"name": "golang", "title": "The Go community", "description": "Gophers welcome",
	})
	if status != http.StatusCreated {
		t.Fatalf("create subreddit: want 201, got %d (%v)", status, got)
	}

Why is the test ~1 second? bcrypt!

Cost 12 hashing takes ~250ms and the test registers + logs in. That's the security feature working as designed. Test suites at big shops inject a lower bcrypt cost in test config — a nice first refactor if the wait annoys you.

Build it, ship it

Go's deployment story is the industry's envy: one static binary, no runtime, no node_modules, no Python venv. And because our SQLite driver is pure Go, cross-compiling to any OS is a two-variable affair:

terminal
# a lean production binary for this machine (-s -w strips debug symbols):
$ go build -ldflags="-s -w" -o gopherit-api ./cmd/api
$ ls -lh gopherit-api
-rwxr-xr-x 1 you you 9.9M gopherit-api

# ...or for a Linux server / a Raspberry Pi / a Mac, from anywhere:
$ GOOS=linux  GOARCH=amd64 go build -o gopherit-linux  ./cmd/api
$ GOOS=linux  GOARCH=arm64 go build -o gopherit-pi     ./cmd/api
$ GOOS=darwin GOARCH=arm64 go build -o gopherit-mac    ./cmd/api

# run it in production mode:
$ ./gopherit-api -port 4000 -env production -dsn /var/lib/gopherit/gopherit.db

Copy that one file to any $5 VPS, run it behind a TLS reverse proxy (Caddy does automatic HTTPS in two config lines), and your Reddit is on the internet. Prefer containers? The repo includes a two-stage Dockerfile that builds a ~12 MB image:

Dockerfile 
FROM golang:1.24 AS build
WORKDIR /src
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -ldflags="-s -w" -o /gopherit ./cmd/api

FROM gcr.io/distroless/static-debian12
COPY --from=build /gopherit /gopherit
EXPOSE 4000
VOLUME /data
ENTRYPOINT ["/gopherit", "-dsn", "/data/gopherit.db", "-env", "production"]
The pre-flight checklist for real traffic

① Serve HTTPS only (the reverse proxy handles it). ② Pin Access-Control-Allow-Origin to your actual frontend's domain — chapter 8's one-line change. ③ Add rate limiting to POST /tokens before someone brute-forces passwords. ④ Back up the .db file (it's just a file — sqlite3 gopherit.db ".backup backup.db").

Plug in a frontend — any frontend

This is the moment the “backend only” promise pays out. Everything a UI needs is three fetch patterns:

any-frontend.js — the whole integration 
const API = "http://localhost:4000/api/v1";

// 1. log in once, keep the token:
const { token } = await (await fetch(`${API}/tokens`, {
  method: "POST",
  body: JSON.stringify({ email, password }),
})).json();

// 2. read things (no auth needed):
const { posts, meta } = await (await fetch(`${API}/posts?sort=hot`)).json();

// 3. do things (token in the header):
await fetch(`${API}/posts/${id}/vote`, {
  method: "POST",
  headers: { Authorization: `Bearer ${token}` },
  body: JSON.stringify({ value: 1 }),
});

Build the UI in React, Vue, Svelte, HTMX, Flutter, SwiftUI, or a curl-powered bash script — the API doesn't know and doesn't care. That's the whole point of the contract you've built.

The whole book on one sticky note 📌

  • HTTP handlers: read r, write w — everything else is arrangement.
  • Layers: handlers speak HTTP, the store speaks SQL, sentinel errors translate between them.
  • Helpers once (readJSON/writeJSON/problems), reused everywhere.
  • Security is defaults: bcrypt, hashed tokens, identical 401s, parameterized SQL, json:"-".
  • The database enforces truth: UNIQUE, CHECK, FKs, composite PKs, transactions.
  • Test the whole app with httptest; ship one static binary.
