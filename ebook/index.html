What you'll have built by the last page

A complete, working Reddit-clone API called gopherit. Users register and log in, create communities (subreddits), submit text & link posts, write nested comment threads, and upvote / downvote everything — with hot/new/top feeds, pagination, auth, CORS and tests. It speaks pure JSON over HTTP, so anyone can bolt a React, Vue, Flutter or terminal frontend onto it and have their own Reddit.

How to read this book

  • Type the code. Every chapter builds one piece of gopherit. Copy-paste works, but your fingers learn faster than your eyes.
  • Run everything. Each chapter ends with real curl commands and the exact output you should see — taped in from a live run of the finished server.
  • Read the sticky notes. Yellow = remember this. The dashed boxes are tips 💡, warnings ⚠️, traps 🪤 and deep dives 🤿 — skippable, but tasty.
  • Stuck? The finished project is in code/gopherit/, compiles cleanly, and passes its test suite. Diff your code against it.

First API ever? Start with the warm-up 🐣

If this is your very first web backend, do the short Build Your First Go API book first — you'll build a tiny link board in an evening. It teaches the core moves on a fraction of the code, and makes this book feel far easier.

Before you start, you need…

  • Go 1.24+ installed — check with go version
  • a terminal with curl (or any API client)
  • basic Go syntax: variables, structs, functions, errors. Rusty? Keep the gocheatsheet open in the next tab — this book was written to be its companion. 📑
No frameworks. No ORMs. No magic. Just Go's standard library, one tiny SQLite driver, and bcrypt. ✍️
