Hello, HTTP

your first Go web server — handlers, routing, path params, and JSON out

In this chapter you'll learn ✍️

  • the two-argument handler signature that rules all Go web code
  • Go 1.22+ routing: "GET /posts/{id}" — methods & params, no framework
  • reading path parameters and query strings
  • sending your first JSON response with struct tags

Fifteen lines to a live server

Make a scratch folder anywhere (this one's a playground — the real project starts next chapter) and drop this in main.go:

hello/main.go 
package main

import (
	"fmt"
	"log"
	"net/http"
)

func main() {
	http.HandleFunc("GET /hello", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "Hello from Go! 🎉")
	})

	log.Println("listening on http://localhost:4000 ...")
	log.Fatal(http.ListenAndServe(":4000", nil))
}
terminal 1
$ go mod init hello && go run .
2026/07/07 15:46:41 listening on http://localhost:4000 ...
terminal 2
$ curl localhost:4000/hello
Hello from Go! 🎉
$ curl -i localhost:4000/nope
HTTP/1.1 404 Not Found

That's a real, concurrent, production-grade HTTP server. Go spins up a goroutine per connection for you — thousands of simultaneous requests, zero extra code. (Goroutines feeling foggy? Your cheatsheet's concurrency page has your back.)

The handler signature — memorize this one

Every piece of Go web code you will ever write is a variation of this function shape:

the shape of all handlers 
func (w http.ResponseWriter, r *http.Request) {
	// r = the request:  everything the client sent  (read from it)
	// w = the response: your pen to write back      (write to it)
}
  • r *http.Request — the opened envelope: method, URL, headers, body. You read from it.
  • w http.ResponseWriter — the reply pen. You set headers, pick a status code, and write bytes. Order matters: headers first, then WriteHeader(status), then the body — once the body starts flowing, headers are sealed.
The classic beginner trap

Calling w.WriteHeader(404) after you've already written body bytes does nothing except log a warning — the 200 already left the building. Write in the right order: headers → status → body.

Routing like it's 2024 (because it is)

Since Go 1.22, the standard router — http.ServeMux — understands methods and path parameters in the pattern string. This is the feature that made half the routing frameworks optional:

routes/main.go 
package main

import (
	"fmt"
	"log"
	"net/http"
)

func main() {
	mux := http.NewServeMux()

	// Method + exact path.
	mux.HandleFunc("GET /posts", listPosts)
	mux.HandleFunc("POST /posts", createPost)

	// {id} is a path parameter — it matches one segment.
	mux.HandleFunc("GET /posts/{id}", getPost)

	log.Println("listening on http://localhost:4000 ...")
	log.Fatal(http.ListenAndServe(":4000", mux))
}

func listPosts(w http.ResponseWriter, r *http.Request) {
	// Query parameters live in the URL after the "?".
	sort := r.URL.Query().Get("sort") // "" when absent
	if sort == "" {
		sort = "hot"
	}
	fmt.Fprintf(w, "a list of posts, sorted by %q\n", sort)
}

func createPost(w http.ResponseWriter, r *http.Request) {
	w.WriteHeader(http.StatusCreated) // 201
	fmt.Fprintln(w, "pretend we created a post")
}

func getPost(w http.ResponseWriter, r *http.Request) {
	id := r.PathValue("id") // whatever matched {id}
	fmt.Fprintf(w, "you asked for post number %s\n", id)
}

Give it a spin:

terminal
$ curl "localhost:4000/posts?sort=top"
a list of posts, sorted by "top"

$ curl localhost:4000/posts/7
you asked for post number 7

$ curl -i -X POST localhost:4000/posts
HTTP/1.1 201 Created

$ curl -i -X DELETE localhost:4000/posts
HTTP/1.1 405 Method Not Allowed
Allow: GET, HEAD, POST

Three things to tattoo somewhere handy:

  • "GET /posts/{id}" — the pattern is method, space, path. {id} matches exactly one path segment, and r.PathValue("id") hands it to you as a string.
  • Query parameters (?sort=top&page=2) are not part of the route. Fish them out with r.URL.Query().Get("sort") — you get "" when absent, so defaults are a one-liner.
  • That 405 with an Allow header? The mux generated it for us — it knows /posts exists but not for DELETE. Frameworks charge for this; std lib does it free.
Why we make our own mux

The hello example used http.HandleFunc, which quietly registers routes on a package-global router (DefaultServeMux) that any imported library can also write to. From now on we always build our own with http.NewServeMux() — explicit, testable, no surprise roommates.

JSON: teaching structs to talk

APIs speak JSON, and Go's encoding/json maps it onto structs. The magic knob is the struct tag — the backtick string that renames fields on their way out:

json/main.go 
package main

import (
	"encoding/json"
	"log"
	"net/http"
	"time"
)

// The struct tags (the bits in backticks) control the JSON key names:
// Title becomes "title", CreatedAt becomes "created_at".
type Post struct {
	ID        int       `json:"id"`
	Title     string    `json:"title"`
	Score     int       `json:"score"`
	CreatedAt time.Time `json:"created_at"`
}

func main() {
	mux := http.NewServeMux()

	mux.HandleFunc("GET /posts/{id}", func(w http.ResponseWriter, r *http.Request) {
		post := Post{
			ID:        1,
			Title:     "Go is pretty nice actually",
			Score:     42,
			CreatedAt: time.Now().UTC(),
		}

		w.Header().Set("Content-Type", "application/json")
		if err := json.NewEncoder(w).Encode(post); err != nil {
			log.Println("encoding failed:", err)
		}
	})

	log.Println("listening on http://localhost:4000 ...")
	log.Fatal(http.ListenAndServe(":4000", mux))
}
terminal
$ curl localhost:4000/posts/1
{"id":1,"title":"Go is pretty nice actually","score":42,"created_at":"2026-07-07T15:46:42Z"}
  • Title string `json:"title"` — exported Go field Title, JSON key title. Only exported (Capitalised) fields are encoded at all.
  • json.NewEncoder(w).Encode(post) streams the JSON straight into the response.
  • Set Content-Type: application/json yourself — otherwise Go guesses text/plain and picky clients sulk.
Tag tricks you'll meet later in this book

json:"body,omitempty" — drop the key entirely when the value is empty (chapter 10, so link posts don't show "body": "").
json:"-" — never serialize this field (chapter 6, for password hashes).
A pointer field like *int64 — encodes as null when nil (chapter 11, for “no parent comment”).

Pin to your brain 📌

  • A handler is func(w http.ResponseWriter, r *http.Request) — read from r, write to w.
  • Route patterns: "METHOD /path/{param}"; grab params with r.PathValue.
  • Query strings via r.URL.Query().Get; absent = "".
  • Struct tags decide JSON key names; unexported fields never leave the building.
  • Always create your own http.NewServeMux().
