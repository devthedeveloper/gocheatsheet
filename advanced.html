Generics

write it once, keep the type safety

Generic functions go 1.18+

func Map[T, U any](
	s []T, f func(T) U) []U {
	out := make([]U, 0, len(s))
	for _, v := range s {
		out = append(out, f(v))
	}
	return out
}

// inference: no [int, int] needed
doubled := Map(nums,
	func(n int) int { return n * 2 })

[T any] declares a type parameter. Callers rarely spell it out — Go infers it from the arguments.

Constraints — what T is allowed to be

type Number interface {
	~int | ~float64 // ~ = named types
}                   //     count too

func Sum[T Number](s []T) T {
	var total T
	for _, v := range s {
		total += v
	}
	return total
}

// stdlib: cmp.Ordered, comparable

Reach for cmp.Ordered (anything with <) and comparable (map-key-able) before writing your own.

Generic types

type Stack[T any] struct {
	items []T
}

func (s *Stack[T]) Push(v T) {
	s.items = append(s.items, v)
}

func (s *Stack[T]) Pop() (T, bool) {
	var zero T
	if len(s.items) == 0 {
		return zero, false
	}
	last := len(s.items) - 1
	v := s.items[last]
	s.items = s.items[:last]
	return v, true
}

st := &Stack[int]{} // instantiate

var zero T is the trick for "return the zero value of whatever T is".

Generics or interfaces?

Same logic over many types (containers, math, slices)→genericsone implementation, compile-time types
Different behavior per type (each store saves differently)→interfaceseach type brings its own method
Only used with one concrete type today→neitherplain code; generalize when it hurts
The stdlib already did it→slices.* and maps.*Contains, Sort, Clone, Index, Keys…

Interfaces, deeper

assertions, custom errors, and the options pattern

Type assertions & type switch

s, ok := val.(string) // safe assert
// without ok: panics if wrong

switch x := val.(type) {
case int:
	// x is an int here
case fmt.Stringer:
	// x.String() available here
case nil:
	// val was nil
default:
	// something else
}

Always use the , ok form unless a wrong type genuinely means a bug worth crashing on.

The two interfaces everyone implements

// fmt.Stringer — how a value prints:
func (n Note) String() string {
	return fmt.Sprintf(
		"#%d %q", n.ID, n.Text)
}
fmt.Println(n) // uses it automatically

// error — it's just an interface:
// type error interface {
//     Error() string
// }

Implement String() on domain types and your logs get readable for free.

Custom error types

type APIError struct {
	Status int
	Msg    string
}

func (e *APIError) Error() string {
	return e.Msg
}

// callers unwrap it by type:
var apiErr *APIError
if errors.As(err, &apiErr) {
	w.WriteHeader(apiErr.Status)
}

Use when errors carry data (status, field name). If it's just a marker, a sentinel errors.New is enough.

Functional options pattern

type Option func(*Server)

func WithPort(p int) Option {
	return func(s *Server) { s.port = p }
}

func WithTLS() Option {
	return func(s *Server) { s.tls = true }
}

func NewServer(opts ...Option) *Server {
	s := &Server{port: 8080} // defaults
	for _, o := range opts {
		o(s)
	}
	return s
}

NewServer(WithPort(9090), WithTLS())

Use for constructors with many optional knobs. Two or three required params? Just use plain arguments.

Concurrency patterns

the shapes that show up in every production service

errgroup — WaitGroup with brains x/sync

g, ctx := errgroup.WithContext(ctx)

for _, url := range urls {
	g.Go(func() error {
		return fetch(ctx, url)
	})
}

if err := g.Wait(); err != nil {
	// first error wins; its ctx
	// cancellation stops the others
}

g.SetLimit(8) // max 8 at once

💡 Think of it: a WaitGroup that also takes complaints. The lead sends N devs off, waits for everyone, and hears the first blocker — and the shared context tells the rest to stop wasting time on a doomed request.

Default choice for fan-out with failures. WaitGroup + error plumbing + cancellation + a concurrency cap, in four lines.

Worker pool

jobs := make(chan Job)
var wg sync.WaitGroup

for range 4 { // exactly 4 workers
	wg.Add(1)
	go func() {
		defer wg.Done()
		for j := range jobs {
			process(j)
		}
	}()
}

for _, j := range work {
	jobs <- j
}
close(jobs) // ends the ranges
wg.Wait()

💡 Think of it: a café hires 4 baristas, not one barista per customer. The queue absorbs the morning rush; the crew size caps the chaos.

Use when work arrives over time or per-item goroutines would be too many. Fixed cost, steady throughput.

Semaphore — cap concurrency

sem := make(chan struct{}, 10)

for _, t := range tasks {
	sem <- struct{}{} // acquire slot
	go func() {
		defer func() { <-sem }()
		run(t)
	}()
}

// struct{} is zero bytes — the
// channel IS the counter

💡 Think of it: a bouncer with 10 wristbands. No wristband, no entry; hand it back on your way out. The buffered channel IS the stack of wristbands.

Same idea as g.SetLimit when you're not using errgroup — e.g. capping calls to a rate-limited API.

sync.Once & atomic

var once sync.Once
once.Do(loadConfig)
// runs exactly once, ever — even
// called from 100 goroutines

// lazy singleton, stdlib-style:
var getDB = sync.OnceValue(connect)

var hits atomic.Int64
hits.Add(1)      // no mutex needed
n := hits.Load()

💡 Think of it: the first person to ask for the wifi password walks to the front desk; everyone after reads it off the whiteboard. Once guarantees exactly one trip to the desk, even if 100 people ask in the same instant.

A lone counter or flag doesn't need a mutex — atomic types are simpler and faster. Anything bigger: mutex.

Channel states — the whole truth

nilopenclosed
ch <-blocks ∞workspanic!
<-chblocks ∞workszero, false

Memorize the two panics: send-on-closed and close-twice. Both mean the same design fix — only the sender closes, exactly once.

The nil-channel trick

// nil channels block forever — so a
// nil case in select is DISABLED:
var tick <-chan time.Time // nil
if verbose {
	t := time.NewTicker(time.Second)
	defer t.Stop()
	tick = t.C
}

select {
case <-tick: // skipped while nil
	logStats()
case v := <-work:
	use(v)
}

💡 Think of it: unplugging a doorbell. The button is still on the wall — the case still compiles — it just never rings while unplugged.

Set a channel variable to nil to switch its case off mid-loop — e.g. after it's drained. No booleans needed.

Which sync tool?

Guarding a shared map / struct→sync.Mutexread-heavy? sync.RWMutex
A single counter or flag→atomic.Int64 / atomic.Boolcheaper than a lock
Passing data, transferring ownership→channel"don't communicate by sharing memory…"
Fan-out where any call can fail→errgroup+ SetLimit for a concurrency cap
One-time initialization→sync.Once / OnceValuelazy singletons without races

How do I stop a goroutine?

A loop doing work→check ctx.Done() each passselect on it alongside the work channel
A worker ranging over a channel→close the channelrange ends when it's drained
From the outside, forcefully→you can'tgoroutines exit only by returning — design the exit in

Context discipline

cancellation is a tree — cut a branch, everything below stops

Cancel & timeout

ctx, cancel := context.WithTimeout(
	ctx, 5*time.Second)
defer cancel() // ALWAYS — even on
               // success (frees timer)

err := slowCall(ctx)
if errors.Is(err,
	context.DeadlineExceeded) {
	// took too long
}
if errors.Is(err, context.Canceled) {
	// someone called cancel()
}

💡 Think of it: an egg timer you set for every request. Whatever's still cooking when it rings gets abandoned — and defer cancel() is putting the timer back in the drawer when you finish early.

Skipping defer cancel() leaks a timer and a goroutine until the deadline — go vet flags it.

Make long work cancellable

for _, item := range hugeList {
	select {
	case <-ctx.Done():
		return ctx.Err()
	default:
	}
	process(item)
}

// the default case makes this a
// non-blocking "peek" at Done

A context only helps if you check it. CPU-bound loops need this peek; blocking calls should take ctx themselves.

Context values — sparingly

type ctxKey struct{} // private key type
                     // = no collisions

ctx = context.WithValue(
	ctx, ctxKey{}, userID)

id, ok := ctx.Value(ctxKey{}).(string)

Only request-scoped data (user ID, trace ID). Never dependencies — a function that needs the DB should take the DB.

The four context rules

func Do(ctx context.Context, …) // 1.
// ctx is ALWAYS the first parameter

type Server struct {
	// ctx context.Context ← 2. NEVER
}   // store one in a struct

context.Background() // 3. only at the
                     // top: main, tests
context.TODO()       // "I'll wire this
                     // up later"

4. Pass it down, never sideways — child work gets a child context, so cancelling the parent stops the whole tree.

Testing muscle

table-driven by default, real HTTP when it matters

Table-driven tests — the Go default

func TestDivide(t *testing.T) {
	tests := []struct {
		name    string
		a, b    float64
		want    float64
		wantErr bool
	}{
		{"simple", 10, 2, 5, false},
		{"by zero", 1, 0, 0, true},
	}
	for _, tt := range tests {
		t.Run(tt.name, func(t *testing.T) {
			got, err := divide(tt.a, tt.b)
			if (err != nil) != tt.wantErr {
				t.Fatalf("err = %v", err)
			}
			if got != tt.want {
				t.Errorf("got %v, want %v",
					got, tt.want)
			}
		})
	}
}

New case = one struct line. t.Run names show up in failures: TestDivide/by_zero. Run one: go test -run 'Divide/zero'.

httptest — test the real API

srv := httptest.NewServer(a.routes())
defer srv.Close()

res, err := http.Get(srv.URL + "/notes")
// real HTTP, random free port,
// same router + middleware as prod

// or skip the network entirely:
w := httptest.NewRecorder()
r := httptest.NewRequest(
	"GET", "/notes/1", nil)
a.routes().ServeHTTP(w, r)
// assert on w.Code, w.Body

This is why handlers take injected deps — swap the store for a fake and the whole API is testable in-memory.

The t.* toolbox

t.Parallel() // this test can run
             // alongside others

t.Cleanup(func() { db.Close() })
// defer, but works across helpers

t.Helper() // failures blame the
           // CALLER's line, not this fn

t.Fatalf("stop this test now: %v", err)
t.Errorf("note failure, keep going")

if testing.Short() {
	t.Skip("skip with: go test -short")
}

Fatal vs Error: can the rest of the test still teach you something? Errorf. Is everything after garbage? Fatalf.

Performance

measure first — then it's usually allocations

Something's slow — in what order do I act?

1. Prove where the time goes→pprof + benchmarksguessing wastes weeks; profiles take minutes
2. Better algorithm / fewer queries→the big wins live herean O(n²) fix beats any micro-tuning
3. Cut allocations in hot paths→prealloc, Builder, Poolallocs/op is the metric to shrink
4. Parallelize→last, and only if measuredgoroutines are not a performance sprinkle

strings.Builder — concat in loops

var b strings.Builder
b.Grow(estimate) // optional prealloc
for _, s := range parts {
	b.WriteString(s)
}
out := b.String()

// s += part in a loop copies the
// whole string every pass — O(n²)

Joining a known slice? strings.Join(parts, ", ") is even simpler — and already optimal.

Preallocate when you know the size

out := make([]Item, 0, len(in))
for _, v := range in {
	out = append(out, convert(v))
}

m := make(map[string]int, count)

// growing from cap 0 = the backing
// array doubles and copies, repeatedly

The single cheapest optimization in Go. If a length is knowable, hand it to make.

sync.Pool — reuse hot objects

var bufs = sync.Pool{
	New: func() any {
		return new(bytes.Buffer)
	},
}

buf := bufs.Get().(*bytes.Buffer)
buf.Reset() // pools return DIRTY items
defer bufs.Put(buf)

// use buf …

💡 Think of it: shared office bikes instead of buying a new bike for every errand — grab one from the rack, ride, put it back. Just adjust the seat first (Reset()): the last rider left it their way.

For big objects allocated per-request (buffers, scratch slices). Measure before and after — pools can lose to the allocator.

pprof — see where it burns

import _ "net/http/pprof"
// registers /debug/pprof/* handlers

// then, from your terminal:
// go tool pprof \
//   localhost:8080/debug/pprof/profile

Never expose /debug/pprof publicly — mount it on an internal-only port in production.

Escape analysis — stack or heap?

go build -gcflags="-m" . 2>&1 \
	| grep escape

// "escapes to heap" = an allocation.
// Common causes: returning pointers
// to locals, interfaces, closures
// capturing, fmt.Println(x)

Don't fight every escape — just know the tool exists for when allocs/op says a hot path allocates and you can't see why.

Tooling

the commands and directives that separate juniors from seniors

The pre-commit four

go test -race ./...  // data races —
                     // run this in CI!
go vet ./...         // static analysis
gofmt -l .           // unformatted?
go mod tidy          // clean go.mod

// races only show on exercised paths:
// -race + weak tests = false comfort

-race found the mutex bugs your code review missed. It's ~10× slower — fine for tests, not for prod builds.

// one binary, no "file not found"
// at 3am because deploy forgot a dir

Templates, migrations, static web assets — embed them and deployment becomes "copy one file".

Build tags & go:generate

// only with:
// go test -tags=integration ./...

//go:generate mockgen -source=store.go

// replace github.com/x/y => ../y

Production patterns

the shapes that keep pagers quiet

Graceful shutdown

ctx, stop := signal.NotifyContext(
	context.Background(),
	os.Interrupt, syscall.SIGTERM)
defer stop()

go srv.ListenAndServe()

<-ctx.Done() // wait for the signal

shCtx, cancel := context.WithTimeout(
	context.Background(), 10*time.Second)
defer cancel()
srv.Shutdown(shCtx) // drain in-flight
                    // requests first

💡 Think of it: closing time at a restaurant — lock the door, let seated guests finish, then lights out. SIGTERM is the "we close in 30 seconds" bell.

Kubernetes sends SIGTERM before killing. This pattern is the difference between clean deploys and dropped requests.

Middleware chains

func chain(h http.Handler,
	mw ...func(http.Handler) http.Handler,
) http.Handler {
	for i := len(mw) - 1; i >= 0; i-- {
		h = mw[i](h)
	}
	return h
}

handler := chain(mux,
	logging, auth, rateLimit)
// requests hit them left → right

Order matters: logging first (sees everything), auth before anything that trusts the user, rate-limit early.

Only retry what's safe to repeat (idempotent). The ctx.Done case means shutdown isn't stuck waiting out your backoff.

singleflight — collapse duplicate calls x/sync

var g singleflight.Group

v, err, _ := g.Do(key,
	func() (any, error) {
		return fetchExpensive(key)
	})

// 100 goroutines ask for the same
// key at once → ONE fetch runs,
// all 100 share its result

💡 Think of it: 100 coworkers ask "what's the wifi password?" in the same second. One person walks to reception; the other 99 wait at their desks and share the answer when it arrives. One trip, 100 answers.

The fix for cache stampedes: when a hot cache key expires, exactly one request rebuilds it instead of all of them.

Gotchas — production grade

each of these has ruined someone's on-call week

😱 The typed-nil trap

func do() *APIError { return nil }

var err error = do()
err == nil // FALSE!

// an interface is (type, value);
// type *APIError is set → not nil.
// fix: functions return `error`,
// never concrete error types

💡 Think of it: an empty envelope is still an envelope. The interface holds (type, value); returning a nil *APIError puts an empty envelope into err — and "is there an envelope?" (err != nil) answers yes.

The #1 Go interview question because it's the #1 real bug. Declare error return types, always.

Goroutine leaks

ch := make(chan int)
go func() { ch <- slow() }()

select {
case v := <-ch:
	return v
case <-ctx.Done():
	return 0 // ← goroutine now blocked
}            //   on ch FOREVER

ch := make(chan int, 1) // buffered:
// sender delivers and exits. fixed.

💡 Think of it: a courier standing at a door forever because the recipient moved out. Every timed-out request strands another courier in the hallway — until the building (your RAM) is full of them.

Every abandoned send is a leak. Rule: if a receiver might give up, buffer the channel by the number of senders.

defer in a loop

for _, f := range files {
	fh, _ := os.Open(f)
	defer fh.Close() // ALL wait for
}                    // function exit 😬

// fix: give each pass its own scope
for _, f := range files {
	func() {
		fh, _ := os.Open(f)
		defer fh.Close() // per pass ✓
		use(fh)
	}()
}

10,000 files = 10,000 open handles held at once. Defer is function-scoped, not block-scoped.

Copying a mutex

type Store struct {
	mu sync.Mutex
	m  map[int]Note
}

func (s Store) Get(id int) Note {
	// ^ value receiver COPIES mu —
	// the lock protects nothing now
	s.mu.Lock()
	…
}

// always pointer receivers on
// mutex-holding types. go vet catches
// this — another reason to run it.

💡 Think of it: photocopying the bathroom key. Now two people each hold "the" key and walk in on each other. A lock only works when there's exactly one of it.

A copied lock is two locks. Same rule for passing structs with mutexes to functions: pointers only.

Shadowed err

var conn *Conn
if useTLS {
	conn, err := dialTLS() // := makes a
	_ = conn               // NEW conn +
	_ = err                // NEW err 😬
}
// outer conn is still nil here!

// fix: declare first, then plain =
conn, err = dialTLS()

:= inside a block redeclares. When assigning to outer variables from an if/for, use = and declare above.

Map values aren't addressable

m := map[string]Note{}
m["a"].Text = "hi" // ✗ compile error

n := m["a"]  // copy out
n.Text = "hi"
m["a"] = n   // put back

// or store pointers:
p := map[string]*Note{}
p["a"].Text = "hi" // ✓ (if non-nil)

Mutating structs in a map a lot? That's the signal to switch to map[K]*V.

JSON: absent vs zero

type Patch struct {
	Count *int `json:"count"`
}

// {}           → Count == nil (absent)
// {"count":0}  → *Count == 0 (sent!)

if p.Count != nil {
	note.Count = *p.Count
}

PATCH endpoints need pointers — a plain int can't tell "didn't send it" from "set it to zero".

Timers in loops

t := time.NewTicker(time.Minute)
defer t.Stop()
for {
	select {
	case <-t.C: // ONE ticker, reused ✓
		flush()
	case v := <-ch:
		use(v)
	}
}
// time.After(…) inside the loop makes
// a fresh timer every iteration

One ticker outside the loop beats a new time.After per pass — and Stop() has a home.
