The one idea

Scaling is measurement. Not architecture diagrams, not “add more servers.” You put the system under real load, watch where it breaks, profile to find the actual bottleneck (almost never the one you'd guess), fix that one thing, and measure again. Every chapter walks this loop with real numbers you can reproduce on your own machine:

load it→ watch it break→ profile→ fix one thing→ measure again↺
